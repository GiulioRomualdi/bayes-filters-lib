The unscented transform for a generic non-\/additive measurement model requires the computation of 2 $\ast$ n + 1 sigma points where n depends not only on the size of the input state but also on the size of the noise. 

While the state size is fixed, the noise size might depend on the number of measurements available, hence n might change online. Since the unscented transform weights, stored in ut\+\_\+weight\+\_\+, depend on n the user can specify, using the ctor input parameter update\+\_\+weights\+\_\+online, that the weights have to be re-\/computed at each iteration.
